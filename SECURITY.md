# Security Policy

## Supported versions

This project is the source for a personal site deployed from `main`. Only the
latest commit on `main` receives security fixes.

| Version    | Supported          |
| ---------- | ------------------ |
| `main`     | :white_check_mark: |
| Older tags | :x:                |

## Reporting a vulnerability

Please **do not** open a public issue for security problems.

Email **frankdonascimentojr@gmail.com** with:

- A description of the vulnerability and its impact
- Steps to reproduce (proof of concept if possible)
- Affected URLs, routes, or commit SHA
- Any suggested remediation

You can expect:

- An acknowledgment within **72 hours**
- An initial assessment within **7 days**
- A fix or mitigation timeline communicated after triage

Responsible disclosure is appreciated — please give a reasonable window for a
fix before any public disclosure.

## Out of scope

- Findings from automated scanners without a working proof of concept
- Issues that require physical access to a user's device
- Social engineering of the maintainer or third parties
- Denial of service via volumetric attacks
